# 提交信息模板（中文）

请在每次提交时保持单一责任：一个提交只做一件事（修复/功能/文档等）。提交信息采用中文，结构如下：

```
<类型>: <简洁描述（7-50 字）>

【可选】详细描述（为什么要改、做了哪些改动、对外部影响、回归/测试说明）

关联 issue/PR: #123
```

常用 `类型` 示例：
- `新增`：添加新功能或模块
- `修复`：修复 bug
- `重构`：重构代码（不改变行为）
- `文档`：仅文档更新
- `格式`：格式化、注释、空白等无功能改动

示例：
- `新增: 添加 LP 反演引擎边界检查`（标题）

  详细描述：
  - 增加边界检查以避免无穷解
  - 新增单元测试覆盖边界情况
  - 已在 `block1-inversion-engine` 分支通过本地测试

- `修复: 修正 README 中的论文链接`（标题）

  详细描述：更新了 README 中的英文/中文论文路径，确保 `paper/main.tex` 与 `paper/main_zh.tex` 链接正确。

建议：可将本文件内容作为团队 `commit.template`：

```bash
git config commit.template docs/COMMIT_MESSAGE_TEMPLATE.md
```
